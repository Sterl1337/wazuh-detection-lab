# Lessons Learned

## What this lab reinforced

- Narrow rules are easier to explain and easier to trust
- Synthetic events make demos safer and more repeatable
- Clear alert naming matters as much as the rule logic
- A small, well-documented setup is better than a sprawling lab
- MITRE mapping helps translate alerts into analyst language

## What to improve next

- Add a fourth rule focused on a different telemetry source
- Add screenshots from the dashboard after replaying each sample
- Add a short tuning note for false-positive reduction
- Add a versioned change log for future lab iterations

## Portfolio takeaway

- The strongest result is not just that the rules trigger.
- The stronger result is that each rule has a clear purpose, a safe sample event, and a readable analyst story.
